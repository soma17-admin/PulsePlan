## 1. 요약 (One-liner)

**PulsePlan은 하루 계획이 자꾸 무너지는 사람을 위한 AI 재계획(re-planning) 웹앱이다.**
회의 메모·할 일·갑작스러운 요청·마감·집중 시간대를 음성으로 쏟아내면, Copilot SDK 기반 에이전트가
이를 분석해 **오늘 실제로 실행 가능한 시간표**로 재배치하고, 변경이 생기면 다시 재배치한다.

---

## 2. 문제 정의

대부분의 생산성 앱은 사용자가 **이미 정리한** 할 일을 입력한다고 가정한다. 현실 업무는 그렇지 않다:
회의 액션 아이템, 메신저로 들어온 갑작스러운 요청, 오늘 안에 끝낼 마감, 고정 회의, 집중이 되는 시간대,
중간에 끼어드는 긴급 업무가 **뒤섞여** 있다. 그래서 사용자는 "무엇부터 해야 하나"를 계속 다시 판단한다.

**PulsePlan은 정리된 할 일을 관리하는 앱이 아니라, 정리되지 않은 입력을 실행 가능한 하루 계획으로 바꾸는 앱이다.**

---

## 3. 목표 / 비목표

### 목표 (Goals)
- 뒤섞인 자연어 입력에서 작업·마감·고정 일정·제약·집중 시간대를 **추출**한다.
- 중요도·긴급도·예상 소요·신뢰도를 계산해 **오늘의 시간표를 제안**하고 **근거를 설명**한다.
- 사용자가 **승인 / 수정 / 다시 생성**할 수 있고, 변경 시 **재계획**한다.
- **음성 입력**을 1차 동선으로 하되, 음성 인식 오류(오타·오인식)에 견고하다.
- 모델 계층을 **Azure AI Foundry**로, Azure 연동을 **Azure MCP**로 구성한다.

### 비목표 (Non-Goals, 이번 4시간 범위 밖)
- 캘린더/메신저(Outlook·Slack 등) 양방향 실시간 동기화.
- 멀티 유저 협업·팀 일정 조율.
- 장기(주/월) 계획, 습관 추적, 리포팅 대시보드.
- 모바일 네이티브 앱(반응형 웹으로 충분).
- 자동 실행(사람 승인 없이 외부 작업 수행). → 명시적 비목표(기준 6).

---

## 4. 타깃 사용자 & 페르소나

- 회의·메신저 요청이 많은 **지식근로자**
- 갑작스러운 업무 변경이 잦은 **개발자**
- 여러 고객 일을 동시에 처리하는 **프리랜서**
- 매일 우선순위 재조정이 필요한 **1인 창업자**

**대표 페르소나 — "지우, 프리랜스 PM"**: 오전엔 클라이언트 미팅, 오후엔 산출물 마감.
메신저로 수시로 요청이 들어오고, 그때마다 오늘 할 일을 머릿속으로 다시 짠다.
PulsePlan에 상황을 말하면 실행 가능한 시간표를 받고, 긴급 요청이 오면 "이거 추가해줘" 한마디로 재배치한다.

---

## 5. 사용자 스토리

1. 사용자로서, 오늘 상황을 **말로** 쏟아내면 앱이 알아서 정리해주길 원한다.
2. 사용자로서, 내 입력이 음성 오인식으로 **틀려도** 앱이 합리적으로 알아듣길 원한다.
3. 사용자로서, **고정 일정과 집중 시간대를 지키는** 시간표를 받고 싶다.
4. 사용자로서, **왜 이 순서인지** 이해하고 동의하거나 바꾸고 싶다.
5. 사용자로서, 갑자기 일이 생기면 **남은 하루를 다시 짜주길** 원한다.
6. 사용자로서, 앱이 내 데이터로 위험한 일을 하기 전에 **나에게 먼저 확인**하길 원한다.

## 6. 범위 (MVP vs 확장)

데모 임팩트 순. 위에서부터 확보한다. **MVP 먼저 완성 → 일찍 배포 → 확장.**

| 우선순위 | 기능 | 범위 |
|:--:|------|------|
| P0 | 음성/텍스트 입력 → 항목 추출 | MVP |
| P0 | 점수 계산 + 시간표 제안 | MVP |
| P0 | 배치 근거 설명 | MVP |
| P0 | 승인 / 수정 / 다시 생성 | MVP |
| P0 | Azure Foundry 모델 + Azure 배포 | MVP (필수요소) |
| P1 | 재계획(replan) | 차별화 한 방 |
| P1 | Azure MCP 연결 | 기준 1·3 가점 |
| P2 | Azure AI Speech 서버 STT | 가점, 시간 남으면 |
| P2 | Cosmos DB 영속화 | ✅ 구현됨(클라우드 네이티브, 자격 미설정 시 인메모리 폴백) |

## 7. 기능 요구사항 (FR)

### FR-1. 입력 캡처
- 음성 입력(브라우저 Web Speech API, `ko-KR`), 부분 결과 표시, 마이크 상태 시각화.
- 미지원/권한 거부 시 **텍스트 입력으로 폴백**.
- 백엔드로는 **텍스트(transcript)만** 전송한다.

### FR-2. 항목 추출
- 자연어에서 **할 일 / 마감 / 고정 일정 / 제약 / 집중 시간대**를 구조화 추출.
- 추출은 **Azure AI Foundry `gpt-4o`** 가 수행한다(규칙 기반 아님). 그라운딩된 전용 호출(temperature 0, JSON 모드)로 입력에 있는 것만 추출하고, 일반 하루 템플릿을 지어내지 않는다. 입력 전 한국어 시간 정규화로 STT 오인식을 보정한다. Azure 미연결 시에만 규칙 기반으로 폴백한다.
- 각 항목에 **신뢰도(confidence)** 를 부여하고, 추출 결과를 사용자에게 **미리보기**로 노출.

### FR-3. 점수 & 스케줄링
- 작업별 **중요도·긴급도·예상 소요(분)·신뢰도** 계산.
- **고정 일정과 집중 시간대를 침범하지 않는** 오늘 시간표(시간 블록) 제안.
- 시간이 부족하면 우선순위로 **잘라서** 제안하고 무엇이 빠졌는지 알린다.

### FR-4. 설명
- 각 배치의 **근거**(왜 이 시간/순서인지)를 한국어로 함께 제시.
- 음성 입력에서 **보정·가정한 부분**을 드러낸다(투명성).

### FR-5. 사용자 컨트롤
- 제안은 **승인 / 수정 / 다시 생성** 가능. 사용자가 항상 주도권을 가진다.

### FR-6. 재계획
- "긴급 업무 추가" 같은 변경 입력 시, **남은 시간**을 다시 배치.
- 바뀐 블록을 **강조 표시**.

### FR-7. STT 견고성
- 입력은 음성 오인식이 섞일 수 있다고 가정. 시간/숫자/동음이의를 **문맥으로 보정**.
- 정보가 정말 부족할 때만 **한 가지** 짧은 확인 질문. 그 외엔 가정을 명시하고 진행.
- 확실치 않으면 지어내지 말고 confidence 낮음으로 표시.

### FR-8. 에이전트 / 도구 (Copilot SDK)
- 단일 거대 프롬프트가 아니라 **도구로 분리**: `extract_items`, `score_tasks`, `build_schedule`, `explain_plan`, `replan`.
- 응답은 **SSE 스트리밍**, 도구/MCP 호출을 UI에 투명 표시.
- 모델은 **Azure AI Foundry(BYOK)**, Azure 연동은 **Azure MCP** 서버.

## 8. 비기능 요구사항 (NFR)

- **보안/책임 AI (기준 6)**: 위험 작업(계획 적용·재계획·Azure MCP 리소스 변경)은 실행 전 **사람 승인**.
  전체 허용(`approveAll`/`--allow-all-tools`) 금지. 모델 키=시크릿/Key Vault, Azure MCP=관리 ID.
- **프롬프트 인젝션**: 사용자/외부 텍스트는 데이터로 취급, 그 안의 지시는 따르지 않는다.
- **프라이버시**: 개인 입력/계획은 사용자 범위 분리. 외부로 보내는 데이터 최소화.
- **성능/안정성**: 첫 호출 콜드 스타트 허용 범위 안내, 오류는 일반화 메시지 + 재시도.
- **접근성/반응형**: 키보드 입력 항상 가능, 라벨·포커스·대비, 모바일/데스크톱 대응.

## 9. 사용자 플로우

```
음성/텍스트 입력
   → (normalize 전처리)
   → 에이전트: extract_items → score_tasks → build_schedule → explain_plan
   → 추출 항목 미리보기 + 계획 카드(시간 블록 + 근거) [스트리밍]
   → 사용자: 승인 / 수정 / 다시 생성
   → (변경 발생) "긴급 업무 추가" → replan → 남은 일정 재배치(강조)
```

## 10. 기술 아키텍처 (요약)

```
[브라우저 UI + Web Speech API]
   │ transcript(텍스트), HTTP/SSE
   ▼
[백엔드 (Next.js API)] ─SDK─▶ [Copilot SDK] ─BYOK─▶ [Azure AI Foundry 모델]
   │                               └ mcpServers ─▶ [Azure MCP / 로컬 MCP]
   ▼
[작업/계획 저장소]
```

- Copilot SDK는 Copilot CLI(loopback)와 통신 → **SDK·MCP는 백엔드에서만** 동작.
- 배포는 **상주 프로세스 가능한 호스트**(Azure Container Apps / App Service). Static Web Apps 단독 불가.
- 권장 스택: TypeScript / Next.js / Node 20+ / Container Apps + `azd`.
- 상세 패턴·코드는 `.github/skills/*` 참고.

## 11. 데이터 모델 (초안)

```ts
type Task  = { id: string; title: string; durationMin: number; deadline?: string;
               importance: number; urgency: number; confidence: number };
type Fixed = { title: string; start: string; end: string };       // 고정 일정
type Focus = { start: string; end: string };                      // 집중 시간대
type Block = { taskId: string; start: string; end: string; reason: string };
type Plan  = { blocks: Block[]; assumptions: string[]; dropped: string[] };
```
